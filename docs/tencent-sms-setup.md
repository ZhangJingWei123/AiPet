# 腾讯云短信（SMS）配置教程（用于手机号验证码登录）

本文面向 `AIPetServer` 的手机号验证码登录，目标是完成：账号开通 → 资质与签名审核 → 模板创建 → API 密钥获取 → 联调验证。

> 截图指引：每一步都标注了建议截图的页面位置与关键字段，按提示截屏即可作为内部留档。

## 0. 前置准备

- 已有腾讯云账号（个人或企业）
- 需要接收验证码的手机号所属国家/地区确定（国内/国际短信不同资质）
- 服务端部署环境具备外网访问（用于调用腾讯云短信接口）

## 1. 注册腾讯云账号与实名认证

1) 打开腾讯云控制台，完成账号注册与登录。

2) 完成实名认证。

- 建议截图：实名认证完成页（展示主体类型与认证通过状态）。

## 2. 开通短信服务

1) 控制台搜索“短信”（SMS），进入短信服务控制台。

2) 若提示未开通，按引导开通服务。

- 建议截图：短信控制台首页（含“应用列表/统计概览”）。

## 3. 创建短信应用（获取 `SdkAppId`）

1) 在短信控制台 → 应用管理（或“应用列表”）→ 新建应用。

2) 记下 `SdkAppId`（后端环境变量 `TENCENT_SMS_APP_ID`）。

- 建议截图：应用详情页（必须包含 `SdkAppId`）。

## 4. 创建短信签名（`SignName`）并提交审核

1) 短信控制台 → 国内短信/国际短信 → 短信签名 → 新建。

2) 签名内容建议：

- 与产品/主体一致，例如：`AIPet` / `AiPet` / `准爸妈同步记录`（以审核规则为准）

3) 按要求上传资质证明材料并提交审核。

- 个人签名一般需要：个人身份信息 + 相关证明（视规则变化）
- 企业签名一般需要：企业资质/商标/授权证明等

- 建议截图：签名列表页（显示签名内容与审核状态）

## 5. 创建短信模板（`TemplateId`）并提交审核

1) 短信控制台 → 短信正文模板 → 新建模板。

2) 验证码模板建议（参数为 `{1}`）：

- 文案示例：`您的验证码为：{1}，5分钟内有效，请勿泄露。`

3) 提交审核，等待状态变为“审核通过”。

- 建议截图：模板详情页（必须包含 `TemplateId`、模板内容、审核状态）。

## 6. 获取 API 密钥（`SecretId` / `SecretKey`）

1) 控制台 → 访问管理（CAM）→ API 密钥管理。

2) 创建密钥，得到 `SecretId` 与 `SecretKey`。

3) 安全建议：

- 仅在服务端环境变量配置，不要写入代码仓库
- 建议创建子账号并最小权限授权（只允许短信相关权限）

- 建议截图：密钥创建成功页（请对 `SecretKey` 打码后留存）。

## 7. 服务端环境变量配置（AIPetServer）

在部署环境设置：

- `TENCENT_SMS_SECRET_ID`
- `TENCENT_SMS_SECRET_KEY`
- `TENCENT_SMS_APP_ID`
- `TENCENT_SMS_SIGN_NAME`
- `TENCENT_SMS_TEMPLATE_ID`
- `TENCENT_SMS_REGION`（可选，默认 `ap-guangzhou`）

### Redis（强烈推荐）

用于验证码存储 + 发送/校验风控。

- `REDIS_ADDR`
- `REDIS_PASSWORD`
- `REDIS_DB`

未配置 Redis 时会退化为内存存储，仅适合本地开发/单实例，生产多实例会导致验证码不一致。

## 8. 联调验证

1) 启动服务端：

```bash
cd AIPetServer
go run main.go
```

2) 发送验证码：

```bash
curl -X POST "http://127.0.0.1:8080/v1/auth/sms/send" \
  -H "Content-Type: application/json" \
  -d '{"phone_number":"+8613800138000"}'
```

3) 校验验证码并登录（用你收到的 `code`）：

```bash
curl -X POST "http://127.0.0.1:8080/v1/auth/sms/verify" \
  -H "Content-Type: application/json" \
  -d '{"phone_number":"+8613800138000","code":"123456"}'
```

4) 预期结果：返回 `access_token`，且 `user` 内包含 `user_uid`（随机 userId）。
